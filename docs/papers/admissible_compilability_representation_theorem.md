# Admissible Compilability: A Representation Theorem for Action Under Approximation

**Core thesis:** an approximate consequential system is admissibly compilable exactly when its ideal licensing map factors through a finite monotone certifier-observable quotient.

---

## Abstract

Approximate consequential systems act on tractable approximations while downstream systems treat their outputs as permission to act. A prior admissibility-compiler discipline proves structural soundness for a fixed implementation: finite ordered permissions, meet-based non-promotion, monotone profiles, exact provenance, immutable registry versions, and runtime downgrade. That result explains why a compiled envelope does not promote beyond its evidence, but it does not characterize when a domain admits such a compiler in the first place.

This paper gives that characterization.

For a claim class `κ`, let `Ωκ` be its action-state space, let `P` be a finite permission chain, and let

```text
λκ : Ωκ → P
```

be the ideal licensing map, sending each state to the strongest permission that would be sound if the ideal state were known. For each permission threshold `p`, define the failure set

```text
F_p = {ω ∈ Ωκ : λκ(ω) < p}.
```

The main theorem proves the equivalence of three conditions, relative to a fixed monotone certifier language `C`:

1. the threshold failure sets `F_p` are finitely generated by `C`-obstruction predicates;
2. the ideal licensing map `λκ` factors through a finite monotone `C`-observable quotient;
3. there exists a bounded, monotone, threshold-complete `C`-compiler for `κ`.

The theorem separates three distinct compiler requirements: boundedness, monotonicity, and sharpness. Boundedness gives finite observation structure. Monotonicity turns denial regions into upward-closed obstruction sets. Sharpness identifies the compiler's denial regions with the ideal failure sets rather than merely conservative over-approximations. A constant-bottom compiler is sound for every domain, but sharp for almost none; therefore soundness alone cannot characterize admissible compilability.

The result also distinguishes schema-finiteness from instance-finiteness. A production system may induce fresh gap instances on every request while still using finitely many obstruction schemas. The theorem is therefore stated for uniformly finite per-judgment quotients rather than a single global finite Boolean cube.

Finally, the paper proves two representative tameness corollaries. In discrete settings, if the worsening preorder is a well-quasi-order, then every upward failure set has a finite minimal obstruction basis. In real-valued settings, if hidden-state compatibility and permission failure are semialgebraic, then Tarski-Seidenberg projection preserves finite definability, though effective production use requires additional smallness assumptions. Composition and expiry dynamics are identified as subsequent theorems: this paper characterizes when the static projection object exists.

## 1. Introduction

An approximate consequential system is one where an unavailable ideal object is replaced by a tractable approximation, and that approximation is then treated as permission to act. Examples include experiment readouts, agent plans, marketplace policy changes, fraud holds, medical triage signals, and cybersecurity responses. The recurring failure is not merely approximation error. It is **mislicensing**: a local score, readout, or certificate is promoted into action authority that it does not support.

The admissibility compiler prevents such promotion by emitting only the strongest permission supported by live evidence, provenance, scope, authority, expiry, runtime context, and structural checks. Its existing proof surface is intentionally small: a finite permission order, finite meets, monotone profiles, exact provenance, immutable registry versions, and runtime downgrade. That proof establishes structural soundness for a compiler implementation.

This paper asks a prior question:

> When does a domain admit such a compiler at all?

The answer cannot be “when a taxonomy can be written.” Taxonomies can be arbitrary. Nor can the answer be “when a sound non-promoting compiler exists.” The constant-bottom compiler that always emits the weakest outcome is sound for every domain, but it explains nothing about the domain's action semantics.

The right object is the **ideal licensing map**

```text
λκ : Ωκ → P,
```

where `Ωκ` is the action-state space for claim class `κ`, and `P` is a finite permission chain. The map `λκ(ω)` is the strongest permission that would be sound at state `ω` if the ideal facts were known.

For each permission threshold `p`, define the corresponding failure set:

```text
F_p = {ω : λκ(ω) < p}.
```

The family `{F_p}` is the mathematical object formerly hidden behind the word “taxonomy.” It names exactly where each permission becomes unsound.

The compiler-shaped domains are precisely those where these threshold failure sets are finitely observable through a monotone certifier language.

Equivalently:

> An approximate consequential system is admissibly compilable exactly when its ideal licensing map factors through a finite monotone certifier-observable quotient.

The paper develops this equivalence and its consequences.

## 2. Relation to the Structural Compiler Paper

This paper is not a replacement for the structural soundness result. It is the representation theorem that explains when the structural result applies sharply.

The structural compiler paper fixes an implementation discipline:

```text
Γ ⊢ z : p until ε
```

where `Γ` is a proof context, `z` is the candidate claim/action, `p` is the emitted permission, and `ε` is an expiry condition. It proves non-promotion under a finite permission order, meet-based composition, exact provenance, fixed registry versions, runtime downgrade, and detail-contract validation.

That result is implementation-relative. It says:

```text
if the compiler's evidence interface is sound,
then the emitted permission is no stronger than the interface supports.
```

This paper asks:

```text
when does the domain's ideal action semantics admit such an evidence interface?
```

The answer is a representation theorem. The compiler works sharply for a class `κ` when the ideal licensing map `λκ` can be observed through a finite monotone quotient generated by certifiable obstructions.

Thus the two results have different roles.

```text
Structural soundness paper:
  Given a compiler discipline, prove it cannot promote.

This paper:
  Characterize when a domain admits a sharp compiler discipline.
```

## 3. Permission Chains and Ideal Licensing

Fix a claim class `κ`.

### Definition 3.1: Permission chain

A **permission chain** is a finite total order

```text
(P, ≤)
```

where lower elements are more restrictive and higher elements are stronger permissions. For example:

```text
OOC ≤ EXP ≤ REF ≤ UNS ≤ ETA ≤ ESC ≤ ROL ≤ DIA ≤ REV ≤ AEX ≤ ALR ≤ AAA.
```

This paper requires only finiteness and total order. The concrete labels are implementation choices.

### Definition 3.2: Action-state space

The **action-state space** for `κ` is a set

```text
Ωκ
```

whose elements contain all facts relevant to the soundness of action under class `κ`. A state may include ideal semantic facts, tractable approximations, candidate identities, contexts, intended use, evidence, authority, scope, and runtime conditions.

The compiler need not observe all of `Ωκ`. In fact, the ideal semantic components are often unavailable. `Ωκ` is the semantic space in which the target licensing map is defined.

### Definition 3.3: Ideal licensing map

The **ideal licensing map** is

```text
λκ : Ωκ → P.
```

`λκ(ω)` is the strongest permission that would be sound at state `ω` if all ideal facts were known.

### Definition 3.4: Permission-threshold failure sets

For each permission threshold `p ∈ P`, define

```text
F_p = {ω ∈ Ωκ : λκ(ω) < p}.
```

Thus `F_p` is the set of states where permission `p` is unsound.

Because `P` is a total order, the failure sets are nested:

```text
p ≤ q  ⇒  F_p ⊆ F_q.
```

Stronger permissions fail in at least as many states as weaker permissions.

## 4. Worsening and Action Monotonicity

A compiler discipline is natural only when “worse” states cannot justify stronger action.

### Definition 4.1: Worsening preorder

A **worsening preorder** on `Ωκ` is a preorder

```text
⪯κ
```

where

```text
ω ⪯κ ω'
```

means that `ω'` is no better than `ω` for action-licensing purposes.

Worsening may include weaker evidence, broader scope, stale dependencies, degraded calibration, missing provenance, greater uncertainty, higher interference, absent authority, or more severe runtime context.

### Axiom 4.2: Action monotonicity

The ideal licensing map is **action-monotone** when

```text
ω ⪯κ ω'  ⇒  λκ(ω') ≤ λκ(ω).
```

If the action state worsens, the strongest sound permission cannot increase.

### Lemma 4.3: Failure sets are upward closed

If `λκ` is action-monotone, then each `F_p` is upward closed under `⪯κ`.

**Proof.** Suppose `ω ∈ F_p` and `ω ⪯κ ω'`. Since `ω ∈ F_p`, `λκ(ω) < p`. By action monotonicity, `λκ(ω') ≤ λκ(ω) < p`. Hence `ω' ∈ F_p`. ∎

Action monotonicity is the semantic load-bearing condition. If a domain violates it, the compiler discipline is not the right abstraction: worsening the state can make stronger permission sound, so failure cannot be represented as monotone obstruction.

## 5. Certifier Languages

The theorem is relative to a fixed certifier language. Without this restriction the statement becomes trivial: if arbitrary predicates can be introduced as atoms, every failure set can be named by fiat. The language must therefore be structured.

### Definition 5.1: Monotone certifier language

A **monotone certifier language** `C` over a class `κ` consists of:

```text
C = (H, Param, Level, Atom, Sem, Induce)
```

where:

1. `H` is a finite set of obstruction schemas.
2. For each schema `h ∈ H`, `Param_h` is a parameter space. It may be infinite.
3. For each schema `h`, `Level_h` is a finite set of required levels or statuses.
4. `Atom` contains instantiated obstruction atoms

   ```text
   a = (h, θ, ℓ)
   ```

   where `h ∈ H`, `θ ∈ Param_h`, and `ℓ ∈ Level_h`.
5. `Sem` assigns each atom an upward-closed subset

   ```text
   ⟦a⟧ ⊆ Ωκ.
   ```

   `ω ∈ ⟦a⟧` means that the obstruction named by `a` is present at state `ω`.
6. `Induce` maps each envelope shape or judgment shape `e` to a finite set of atoms

   ```text
   I_e ⊂ Atom.
   ```

The language is **schema-finite** because `H` is finite. It is **instance-finite per judgment** because `I_e` is finite. It may have infinitely many possible atom instances across all judgments.

### Definition 5.2: Positive `C`-formula over a judgment shape

Given a judgment shape `e`, a positive `C`-formula over `e` is generated from atoms in `I_e` using finite conjunction and finite disjunction:

```text
φ ::= a | φ ∧ φ | φ ∨ φ.
```

Negation is not primitive in obstruction formulas. Absence of obstruction must be certified by a positive witness in the evidence system, not by treating failure to observe an atom as proof of safety.

### Definition 5.3: Observation map

For each judgment shape `e`, define the finite Boolean obstruction cube

```text
B_e = {0,1}^{I_e}.
```

The observation map is

```text
α_e : Ω_e → B_e
```

where

```text
α_e(ω)_a = 1  iff  ω ∈ ⟦a⟧.
```

The cube is ordered coordinatewise:

```text
v ⊑ w  iff  ∀a ∈ I_e. v_a ≤ w_a.
```

Thus `w` is at least as obstructed as `v`.

Because each atom is upward closed, `α_e` is monotone with respect to worsening:

```text
ω ⪯κ ω'  ⇒  α_e(ω) ⊑ α_e(ω').
```

## 6. Finite Obstruction Bases

### Definition 6.1: Local threshold failure set

For a judgment shape `e`, define

```text
Ω_e ⊆ Ωκ
```

as the set of states compatible with that shape. The local threshold failure set is

```text
F_{e,p} = F_p ∩ Ω_e.
```

### Definition 6.2: Finite `C`-obstruction basis

The class `κ` admits a **finite `C`-obstruction basis** if, for every judgment shape `e` and permission threshold `p`, there exists an upward subset

```text
U_{e,p} ⊆ B_e
```

such that

```text
F_{e,p} = α_e^{-1}(U_{e,p}).
```

Equivalently, since `B_e` is finite, there exists a positive `C`-formula `φ_{e,p}` over `I_e` such that

```text
ω ∈ F_{e,p}  iff  φ_{e,p}(α_e(ω)) = true.
```

This definition separates global schema finiteness from per-judgment instance finiteness. A deployment can see new claims, candidates, tokens, or provenance atoms on every request, but each request still induces only a finite obstruction cube from finitely many schemas.

### Lemma 6.3: Upward subsets of finite cubes have finite positive bases

Let `B = {0,1}^I` be a finite Boolean cube with coordinatewise order. If `U ⊆ B` is upward closed, then

```text
U = ↑M
```

where `M` is the finite antichain of minimal elements of `U`. Moreover, `U` is expressed by the positive formula

```text
∨_{m ∈ M} ∧_{a : m_a = 1} a.
```

**Proof.** Since `B` is finite, every nonempty subset has finitely many minimal elements. Upward closure gives `U = ↑Min(U)`. The displayed formula holds exactly on valuations that dominate at least one minimal true valuation. ∎

## 7. Bounded, Monotone, Sharp Compilers

A compiler object has three logically distinct properties. They should not be hidden inside one adjective.

### Definition 7.1: Bounded observational compiler

A compiler `K` over `C` is **bounded observational** if, for every judgment shape `e`, there is a finite query set `I_e` and a function

```text
q_e : B_e → P
```

such that

```text
K_e = q_e ∘ α_e.
```

Boundedness is the finite-query condition. The compiler observes only finitely many `C`-atoms per judgment.

### Definition 7.2: Quotient monotonicity

A bounded observational compiler is **quotient-monotone** if each `q_e` is antitone from obstruction valuations to permissions:

```text
v ⊑ w  ⇒  q_e(w) ≤ q_e(v).
```

Adding obstructions cannot increase emitted permission.

### Definition 7.3: Semantic monotonicity

A compiler is **semantically monotone** if

```text
ω ⪯κ ω'  ⇒  K_e(ω') ≤ K_e(ω).
```

If `α_e` is monotone and `q_e` is quotient-monotone, then `K_e` is semantically monotone.

### Definition 7.4: Threshold completeness / sharpness

A compiler is **threshold-complete** or **sharp** for `λκ` when, for every `ω ∈ Ω_e` and every permission `p ∈ P`,

```text
K_e(ω) ≥ p  iff  λκ(ω) ≥ p.
```

Since `P` is a finite chain, this is equivalent to

```text
K_e(ω) = λκ(ω).
```

Sharpness is stronger than soundness. A sound compiler only requires

```text
K_e(ω) ≤ λκ(ω).
```

The constant-bottom compiler is sound for every `κ`; it is sharp only when `λκ` is constantly bottom. Therefore soundness alone cannot characterize admissible compilability.

### Definition 7.5: Bounded sharp monotone `C`-compiler

A **bounded sharp monotone `C`-compiler** for `κ` is a compiler that is:

1. bounded observational;
2. quotient-monotone;
3. threshold-complete for `λκ`.

Semantic monotonicity follows from the first two conditions, provided `C`-atoms are monotone.

## 8. Main Representation Theorem

### Theorem 8.1: Admissible Compilability Representation Theorem

Fix a claim class `κ`, a finite permission chain `P`, an action-state space `Ωκ`, an action-monotone ideal licensing map

```text
λκ : Ωκ → P,
```

and a monotone certifier language `C`.

For each judgment shape `e`, let `Ω_e ⊆ Ωκ`, let `I_e` be the finite set of induced `C`-atoms, and let

```text
α_e : Ω_e → B_e = {0,1}^{I_e}
```

be the obstruction observation map.

The following are equivalent:

**(A) Finite obstruction basis.**  
For every judgment shape `e` and permission threshold `p`, the threshold failure set

```text
F_{e,p} = {ω ∈ Ω_e : λκ(ω) < p}
```

is the inverse image under `α_e` of an upward subset of the finite obstruction cube:

```text
F_{e,p} = α_e^{-1}(U_{e,p})
```

for some upward `U_{e,p} ⊆ B_e`.

**(B) Finite monotone observable quotient.**  
For every judgment shape `e`, the ideal licensing map factors as

```text
Ω_e --α_e--> B_e --q_e--> P
```

where `q_e` is quotient-monotone:

```text
v ⊑ w  ⇒  q_e(w) ≤ q_e(v),
```

and

```text
λκ|_{Ω_e} = q_e ∘ α_e.
```

**(C) Bounded sharp monotone compiler.**  
There exists a bounded observational, quotient-monotone, threshold-complete `C`-compiler for `κ`.

### Proof

#### `(A) ⇒ (B)`

Assume finite obstruction bases.

For each judgment shape `e` and threshold `p`, let

```text
U_{e,p} ⊆ B_e
```

be the upward subset satisfying

```text
F_{e,p} = α_e^{-1}(U_{e,p}).
```

Define

```text
q_e(v) = max { p ∈ P : v ∉ U_{e,p} }.
```

Equivalently, `q_e(v)` is the strongest permission whose failure condition is not triggered by valuation `v`.

The set is nonempty because the bottom permission has empty or vacuous failure threshold; if necessary, adjoin a bottom element `⊥` that is always permitted. Since `P` is finite, the maximum exists.

Now take `ω ∈ Ω_e`. For any `p`,

```text
q_e(α_e(ω)) < p
```

iff the valuation `α_e(ω)` lies in the failure region for threshold `p`, iff

```text
ω ∈ F_{e,p},
```

iff

```text
λκ(ω) < p.
```

Since this equivalence holds for every threshold `p` in a finite chain,

```text
q_e(α_e(ω)) = λκ(ω).
```

Thus `λκ|_{Ω_e} = q_e ∘ α_e`.

It remains to show quotient monotonicity. Suppose `v ⊑ w`. If `q_e(v) = r`, then every permission strictly above `r` fails at `v`, and no permission at or below `r` is ruled out by the threshold family. Since each `U_{e,p}` is upward, any failure triggered at `v` is also triggered at `w`. Therefore `w` cannot support any permission above `r`. Hence

```text
q_e(w) ≤ q_e(v).
```

So the factorization is finite, observable, and monotone.

#### `(B) ⇒ (C)`

Assume finite monotone observable quotient factorizations.

Define the compiler for shape `e` by

```text
K_e(ω) = q_e(α_e(ω)).
```

It is bounded observational because `α_e` queries only the finite set `I_e`. It is quotient-monotone because `q_e` is quotient-monotone. It is threshold-complete because

```text
K_e(ω) = q_e(α_e(ω)) = λκ(ω).
```

Thus a bounded sharp monotone `C`-compiler exists.

#### `(C) ⇒ (A)`

Assume a bounded observational, quotient-monotone, threshold-complete `C`-compiler exists.

By bounded observationality, for each judgment shape `e`, there is a finite obstruction cube `B_e` and a function

```text
q_e : B_e → P
```

such that

```text
K_e = q_e ∘ α_e.
```

For each threshold `p`, define

```text
U_{e,p} = {v ∈ B_e : q_e(v) < p}.
```

Since `q_e` is quotient-monotone, `U_{e,p}` is upward closed. Indeed, if `v ∈ U_{e,p}` and `v ⊑ w`, then

```text
q_e(w) ≤ q_e(v) < p,
```

so `w ∈ U_{e,p}`.

By threshold completeness,

```text
K_e(ω) < p  iff  λκ(ω) < p.
```

Therefore

```text
F_{e,p}
  = {ω : λκ(ω) < p}
  = {ω : K_e(ω) < p}
  = {ω : q_e(α_e(ω)) < p}
  = α_e^{-1}(U_{e,p}).
```

Thus each `F_{e,p}` is generated by an upward subset of the finite obstruction cube. By Lemma 6.3, it has a finite positive obstruction formula. ∎

## 9. The Canonical Quotient

The representation theorem proves existence. There is also a canonical minimal quotient, in the sense of coarsest exact permission quotient.

### Definition 9.1: Exact factorization

An exact factorization of `λ_e = λκ|_{Ω_e}` is a diagram

```text
Ω_e --β--> Q --r--> P
```

such that

```text
λ_e = r ∘ β.
```

We ignore unreachable elements of `Q`, replacing `Q` by `β(Ω_e)` when necessary.

### Proposition 9.2: Coarsest exact quotient

The image of the ideal licensing map

```text
Q_e^min = im(λ_e) ⊆ P
```

with

```text
β_min = λ_e : Ω_e → Q_e^min
```

and inclusion

```text
i : Q_e^min ↪ P
```

is the coarsest exact quotient of `λ_e`.

For any exact factorization

```text
Ω_e --β--> Q --r--> P,
```

there is a unique map on reachable quotient states

```text
h : Q → Q_e^min
```

given by

```text
h(β(ω)) = λ_e(ω)
```

such that

```text
β_min = h ∘ β.
```

### Proof

Well-definedness follows because if `β(ω)=β(ω')`, then

```text
λ_e(ω)=r(β(ω))=r(β(ω'))=λ_e(ω').
```

The equation `β_min = h ∘ β` follows immediately. Uniqueness holds because reachable quotient states are of the form `β(ω)`. ∎

### Corollary 9.3: Minimal sharp compiler quotient

If `λ_e` is `C`-observable, then `Q_e^min = im(λ_e)` is the unique coarsest sharp `C`-observable quotient, up to order-isomorphism and unreachable-state deletion.

This is the precise sense in which the compiler is canonical: any sharp compiler computes at least the information needed to determine `λ_e`; the coarsest such quotient is just the permission value itself. Obstruction bases give finite observable presentations of that quotient.

### Remark 9.4: Minimal obstruction bases need extra structure

The coarsest permission quotient is canonical. A coordinate-level obstruction basis need not be unique unless the certifier language supplies a canonical decomposition notion: for example minimal bad states in a well-quasi-order, prime components in a Noetherian setting, or minimal true valuations in a fixed Boolean cube.

Thus there are two different minimality claims:

```text
coarsest exact permission quotient: canonical generally;
minimal obstruction coordinates: canonical only relative to decomposition structure.
```

## 10. Schema-Finiteness Versus Instance-Finiteness

A common deployment objection is:

> “My system sees new gap instances on every request. How can the obstruction basis be finite?”

The answer is that the theorem requires finite schemas globally and finite instances per judgment, not a single global finite atom set.

### Example 10.1: Provenance atoms with infinite identifiers

Let the language have a finite schema

```text
provenance_missing(τ, g, c, z, x)
```

where the parameters are token id, gap id, claim id, candidate id, and context id.

The parameter space is infinite because new tokens, claims, candidates, and contexts appear over time. But a single judgment envelope contains only finitely many relevant gaps and tokens. Therefore the induced atom set

```text
I_e = {
  provenance_missing(τ, g, c, z, x)
  for τ,g relevant to envelope e
}
```

is finite.

The local observation map is therefore finite:

```text
α_e : Ω_e → {0,1}^{I_e}.
```

Across requests, the cube changes identity. The compiler is not one global Boolean function over all possible IDs. It is a uniformly presentable family of finite Boolean functions indexed by judgment shape.

This matches production reality:

```text
finite schemas across the system;
finite induced instances per judgment;
unbounded fresh instances over the lifetime of the system.
```

### Example 10.2: Marketplace rollout gaps

A marketplace policy-change class may use a finite schema set:

```text
calibration_gap
proxy_gap
freshness_gap
interference_gap
coupling_gap
authority_gap
scope_gap
```

But each policy candidate, placement, market, experiment window, and rollout context induces distinct parameterized instances. The representation theorem applies to the finite per-envelope quotient induced by the candidate/context pair, not to a single global list of all possible marketplace gap instances.

## 11. Static Expiry as an Obstruction Atom

The judgment form includes an expiry condition:

```text
Γ ⊢ z : p until ε.
```

At the static level, expiry is just another monotone obstruction atom.

Define

```text
expired(ε, ρ)
```

to be true when the expiry condition has fired under runtime context `ρ`. The atom is monotone under worsening runtime context when worsening includes forward time, stale dependency discovery, revocation, missing runtime values, or fired expiry rules.

Thus expiry fits the representation theorem:

```text
expired(ε,ρ) = 1  ⇒  permission threshold above EXP fails.
```

This does not prove the dynamical theorem for expiry. It only says that static expiry can be absorbed into the obstruction language without changing the representation theorem.

The dynamic theorem would ask for a family of runtime operators

```text
R_t : Envelope → Envelope
```

satisfying semigroup or killed-process laws. That requires additional structure: evidence aging, token revocation dynamics, context drift, and a time-evolution law. The static theorem says expiry can be handled. The dynamic theorem would explain why expiry must be handled through a monotone downgrade family.

## 12. Corollary: Well-Quasi-Ordered Discrete Domains

The cleanest discrete sufficient condition is a well-quasi-order.

### Definition 12.1: Well-quasi-order

A preorder `(X, ⪯)` is a **well-quasi-order** when every infinite sequence

```text
x_0, x_1, x_2, ...
```

contains indices `i < j` such that

```text
x_i ⪯ x_j.
```

Equivalently, there are no infinite descending sequences and no infinite antichains.

### Theorem 12.2: WQO finite obstruction corollary

Let `(Ωκ, ⪯κ)` be a well-quasi-order. Let `P` be a finite permission chain. Let

```text
λκ : Ωκ → P
```

be action-monotone.

Then every threshold failure set

```text
F_p = {ω : λκ(ω) < p}
```

has a finite minimal basis:

```text
F_p = ↑{m_{p,1}, ..., m_{p,n_p}}.
```

If the certifier language `C_WQO` contains principal-upset obstruction atoms

```text
bad_at_least(m) := {ω : m ⪯κ ω}
```

for certifiable minimal bad states `m`, then `κ` admits a finite `C_WQO`-obstruction basis and hence a bounded sharp monotone compiler.

### Proof

By Lemma 4.3, each `F_p` is upward closed. In a well-quasi-order, every upward-closed set has finitely many minimal elements; otherwise the minimal elements would form an infinite antichain. Therefore

```text
F_p = ↑Min(F_p)
```

with `Min(F_p)` finite. The principal-upset atoms for these minimal elements form a finite obstruction basis. The main representation theorem then gives the compiler. ∎

### Effective caveat

The WQO result gives existence. A production compiler also needs an effective way to compute or certify the minimal bad basis. That may require additional algorithmic structure: decidable ordering, effective predecessor bases, or domain-specific finite contracts.

## 13. Corollary: Semialgebraic Approximation Domains

Real-valued approximation systems naturally involve inequalities: calibration error below a threshold, freshness age below a limit, interference bound below a tolerance, blast radius below a ceiling, or authority ceiling above a requested permission.

This suggests a semialgebraic route.

### Setup 13.1

Suppose the visible state variables live in `R^n`, hidden ideal variables live in `R^m`, and the compatibility relation between ideal and visible states is semialgebraic:

```text
Compat(s, v).
```

Suppose each permission failure condition is semialgebraic in the combined variables:

```text
Fail_p(s, v).
```

The visible threshold failure set is

```text
F_p^vis = { v : ∃s. Compat(s,v) ∧ Fail_p(s,v) }.
```

### Theorem 13.2: Semialgebraic finite-definability corollary

If `Compat` and `Fail_p` are semialgebraic, then `F_p^vis` is semialgebraic. Therefore it has a finite Boolean description by polynomial equalities and inequalities. If the certifier language contains the relevant polynomial-inequality atoms and the failure sets are action-monotone, then the class admits a finite semialgebraic obstruction presentation.

### Proof

Semialgebraic sets are closed under finite Boolean operations. The set

```text
{(s,v) : Compat(s,v) ∧ Fail_p(s,v)}
```

is semialgebraic. The visible failure set `F_p^vis` is its projection along the hidden ideal coordinates `s`. By the Tarski-Seidenberg theorem, projections of semialgebraic sets are semialgebraic. A semialgebraic set has a finite formula using polynomial equalities and inequalities. Under action monotonicity, the resulting threshold regions are upward failure regions. The representation theorem applies once those formula pieces are certifier-realizable. ∎

### Effective caveat

The corollary proves finite definability, not practical smallness. Quantifier elimination for real closed fields can have doubly exponential worst-case complexity. Production use therefore needs additional effective hypotheses, such as:

```text
bounded dimension;
bounded polynomial degree;
sparse interaction structure;
restricted projection form;
contract-specific certificates;
known convex or polyhedral reductions;
small witness families.
```

Without such assumptions, the obstruction basis may exist mathematically but be too large to induce, certify, or audit.

## 14. Other Tameness Regimes

This paper proves the representation theorem and two template corollaries. Other geometric settings should be developed in a separate paper, but the expected pattern is now clear.

### Noetherian route

If threshold failure sets are closed or constructible in a Noetherian space, then finiteness follows from descending-chain conditions and finite irreducible decomposition. Obstructions correspond to generators, components, or strata of the failure locus.

### O-minimal route

If the action-state space and failure relations are definable in an o-minimal structure, then definable sets admit finite cell decompositions. Obstruction bases may be extracted from the finite cell or stratification structure, subject again to effective size bounds.

### Tropical / polyhedral route

If the native approximation geometry degenerates or tropicalizes to a finite polyhedral complex, then obstruction regions may be represented by cones, faces, or systems of linear inequalities. The compiler quotient is then a finite combinatorial shadow of richer geometry.

These regimes share the same meta-principle:

```text
finite permission thresholds
+ monotone failure
+ tame projected geometry
+ certifier-realizable finite presentations
⇒ admissible compilability.
```

## 15. Sound but Incomplete Compilers

The main theorem uses sharpness. It is useful to record what remains true under soundness alone.

### Definition 15.1: Sound compiler

A compiler `K` is sound when

```text
K_e(ω) ≤ λκ(ω)
```

for all states `ω`.

### Proposition 15.2: Sound compilers generate conservative supersets

For each threshold `p`, define the compiler denial region

```text
D_{e,p} = {ω : K_e(ω) < p}.
```

If `K` is sound, then

```text
F_{e,p} ⊆ D_{e,p}.
```

### Proof

If `ω ∈ F_{e,p}`, then `λκ(ω) < p`. Since `K_e(ω) ≤ λκ(ω)`, we have `K_e(ω) < p`. Hence `ω ∈ D_{e,p}`. ∎

Thus a sound compiler may deny many states where permission would ideally be sound. It provides a finite conservative over-approximation of the true failure sets, not a representation of them. Sharpness is exactly the condition that removes the extra conservatism:

```text
D_{e,p} = F_{e,p}.
```

This is why the constant-bottom compiler is not evidence of admissible compilability.

## 16. Composition as the Next Theorem

The representation theorem is static. It identifies when the projection object exists. It does not yet prove when compositions of evidence or envelopes are exact.

Let `E` be an evidence algebra with operation

```text
⊗
```

and let

```text
α : E → B
```

be an obstruction abstraction.

For issued envelopes, conservative composition should satisfy a lax monoidal law in the obstruction order:

```text
α(E_1 ⊗ E_2) ⊒ α(E_1) ∨ α(E_2),
```

where `⊒` means “at least as obstructed.” This says composition cannot hide an obstruction present in a component.

Exact composition would require equality:

```text
α(E_1 ⊗ E_2) = α(E_1) ∨ α(E_2).
```

That condition is much stronger and generally false when raw evidence components are complementary. One evidence object may close calibration while another closes interference; recompiling from the union can produce a stronger permission than composing the two already-issued envelopes.

This suggests a second theorem:

```text
finite obstruction basis
+ lax monoidal obstruction abstraction
⇒ safe envelope composition

finite obstruction basis
+ strong monoidal obstruction abstraction
⇒ exact composition
```

The expected production pattern is lax-but-not-strong. The gap between raw evidence recombination and envelope composition is the operational signature of evidence laundering: a fresh component can be used to obscure a stale or narrow one unless composition is forced through meet/intersection/min-expiry rules.

This is layer 4 work. It is not part of the static representation theorem.

## 17. Expiry Dynamics as the Next Theorem

The static theorem treats expiry as a monotone obstruction atom. That is sufficient for non-promotion:

```text
expired(ε,ρ) ⇒ deny thresholds above EXP.
```

A dynamic theorem would require more. It would define a family of runtime operators

```text
R_t : Envelope → Envelope
```

and ask for laws such as:

```text
R_0 = id
R_{s+t} = R_s ∘ R_t
R_t(E).permission ≤ E.permission.
```

In stochastic settings, evidence aging and revocation may produce a killed semigroup or path-space projection. That structure is not implied by finite obstruction bases alone. It requires a model of time evolution for evidence, context, authority, and expiry rules.

Thus this paper gives the prerequisite:

```text
when static projection into a permission quotient is possible.
```

The expiry-dynamics paper would explain:

```text
how that quotient evolves over time.
```

## 18. Discussion: Why the Class Is Not Fiddly

The characterization is not domain-by-domain. It is structural.

A class is admissibly compilable when the following diagram exists:

```text
Ωκ --α--> finite obstruction quotient --q--> P
```

with

```text
λκ = q ∘ α.
```

This is why the compiler implementation is clean. Once the finite quotient exists, the enforcement algebra is forced:

```text
observe obstruction status;
map status to strongest unblocked permission;
meet with structural and runtime blockers;
never upgrade at runtime.
```

The hard domain question is not whether one can write a taxonomy. The hard question is whether the domain's ideal action-unsoundness has a finite, monotone, certifier-realizable presentation.

Broad families satisfy that condition when they have some tameness principle:

```text
WQO:          finite minimal bad patterns;
Noetherian:   finite components / finitely generated ideals;
Semialgebraic: finite inequality descriptions stable under projection;
O-minimal:    finite cell decomposition;
Tropical:     finite polyhedral shadow.
```

The common structure is:

```text
monotone threshold failure
+ finite/tame presentation
⇒ finite obstruction basis
⇒ admissibility compiler.
```

That is the structural truth behind the compiler discipline.

## 19. Conclusion

The compiler works sharply for a domain exactly when the domain's ideal licensing map factors through a finite monotone certifier-observable quotient.

Equivalently, each permission-threshold failure set must be a finitely generated upward set in the certifier language.

This gives the missing class boundary:

```text
approximate consequential system
+ action monotonicity
+ finite/tame certifier-observable failure theory
= admissibly compilable system.
```

The result explains why the structural compiler proof is so small. The implementation is not discovering domain truth. It is evaluating a finite monotone quotient of that truth.

The next mathematical steps are separate:

1. prove composition theorems for lax versus strong monoidal evidence algebras;
2. prove expiry dynamics theorems for runtime downgrade families;
3. develop geometric-family corollaries beyond WQO and semialgebraic settings;
4. identify effective smallness conditions that make finite bases deployable, not merely existent.
